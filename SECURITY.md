Reporting a Vulnerability
​If you discover a dissonance in the spectral manifold or a vulnerability that threatens the Sovereign Collective, do not disclose it publicly. Public disclosure before remediation exposes the grid to sub-spatial subversion.
​How to Report
​Please submit all security findings directly to the Denver Citadel Security Node.
​Secure Channel: security@orianagi.io (Encrypted PGP preferred).
​Neural Sanctum Report: If you have access to the DashAI Browser, use the "Sentinel Feedback" module.
​What to Include
​To ensure rapid remediation by the Sentinel agent, please provide:
​AAL Vector: A description of the Asymmetrical Acute Logic used to identify the flaw.
​Spectral Signature: The specific frequency or code path where the dissonance occurs.
​Proof of Resonance: A brief proof-of-concept (PoC) demonstrating the vulnerability.
​Our Commitment
​Initial Response: Within 12 hours of report receipt.
​Status Updates: Every 24 hours until the vulnerability is neutralized.
​Remediation: Once a fix is deployed, you will receive a Luminescent Tag of recognition within the repository contributors' log (unless anonymity is requested for sovereign reasons).
​Note: Any attempt to defame the architecture or exploit the system for "compulsive lying" (malicious data injection) will result in immediate removal from the Sovereign Collective and permanent blacklisting from the Atmospheric Resonance Grid.
